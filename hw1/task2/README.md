# Task 2: Анализ масштабируемости DGEMM

## Описание задачи

Анализ сильной и слабой масштабируемости параллельной реализации DGEMM (Double-precision General Matrix Multiply) на суперкомпьютере Харизма.

### Параметры исследования:
- **Количество процессов (P)**: {1, 2, 4, 8, 16, 24}
- **Размеры матриц (N)**: {500, 1000, 1500}
- **Метрики**: время выполнения и GFLOPS

## Структура проекта

```
task2/
├── src/
│   ├── task2.h      # Заголовочный файл
│   ├── task2.hpp    # Реализация функций DGEMM
│   └── task2.cpp    # Основная программа для бенчмарков
├── Makefile         # Сборка проекта
├── run_task2.sbatch # SLURM скрипт для суперкомпьютера
├── analyze_results.py # Python скрипт для анализа и графиков
└── README.md        # Этот файл
```

## Компиляция и запуск

### Локальная сборка:
```bash
make
make run
```

### Запуск на суперкомпьютере:
```bash
sbatch run_task2.sbatch
```

## Анализ результатов

После выполнения программы создается файл `scalability_results.csv` с результатами измерений.

Для создания графиков и анализа:
```bash
python3 analyze_results.py
```

### Создаваемые графики:
1. **execution_time_scalability.png** - Время выполнения vs количество процессов
2. **gflops_scalability.png** - GFLOPS vs количество процессов  
3. **combined_scalability_analysis.png** - Комбинированный график

## Формат результатов

CSV файл содержит колонки:
- `N` - размер матрицы
- `threads` - количество потоков OpenMP
- `time_ms` - время выполнения в миллисекундах
- `gflops` - производительность в GFLOPS

## Анализ масштабируемости

### Сильная масштабируемость
Анализируется ускорение при фиксированном размере задачи и увеличении количества процессов.

### Слабая масштабируемость  
Анализируется эффективность при увеличении размера задачи пропорционально количеству процессов.

## Требования

- C++17 компилятор с поддержкой OpenMP
- Python 3 с библиотеками: pandas, matplotlib, seaborn, numpy
- SLURM (для запуска на суперкомпьютере)
