# Task 3: Shared Memory Bank Conflict Optimization

## Описание

Исследование оптимизации банков разделяемой памяти. Сравнение производительности версий с padding и без padding для избежания bank conflicts.

## Проблема bank conflicts

Shared memory в CUDA организована в 32 банка. Когда несколько потоков обращаются к одному и тому же банку одновременно, возникают конфликты (bank conflicts), которые снижают производительность.

## Решение

Добавление padding (дополнительного столбца) к массивам в shared memory:
- Без padding: `__shared__ double tile[16][16]` - возможны конфликты
- С padding: `__shared__ double tile[16][17]` - конфликты устранены

## Компиляция и запуск

```bash
module load nvidia_sdk/nvhpc/23.5
make
./gemm [N]
```

## Результаты

Программа сравнивает две версии:
1. Без padding (потенциальные bank conflicts)
2. С padding (избежание bank conflicts)

Выводится время выполнения, GFLOPS и speedup для анализа эффективности оптимизации.

## Анализ

- При наличии bank conflicts padding должен улучшить производительность
- Overhead от дополнительной памяти обычно минимален
- Эффект зависит от паттерна доступа к памяти
